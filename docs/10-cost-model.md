## 4. LLM consumption economics behind row 10.1

Per-screening prompt as implemented (`AnthropicAiProvider`): documents truncated
to 24K chars ≈ ~6K tokens + system/context ≈ **~7K input / ~1–2K output tokens**.

| Cost input | Rate (2026-07) | Per screening |
|---|---|---|
| Claude Sonnet (configured default) | $3 in / $15 out per MTok | **≈ $0.04–0.05** |
| Claude Haiku 4.5 (cost-reduced option) | $1 in / $5 out per MTok | ≈ $0.015 |
| Embeddings (self-hosted TEI sidecar) | infra only | $0 marginal (in 11.1) |

Raw spend at 10,000 screenings/month on Sonnet ≈ **$450/mo**. A sell price of
**$0.25–0.50 per screening** (or equivalent monthly blocks) covers retries,
re-screens, model-version headroom, and monitoring while keeping the scored cost
low. Batch-API discounts don't apply (interactive latency); prompt caching is
immaterial at this prompt shape (variable content dominates the prefix).

## 7. Invoice checklist (per §6.2 + DEAF)

Every invoice: contract number · BTRC number · milestone number + deliverable
name · **signed DEAF reference** · amount, withhold shown, net payable ·
remit-to address exactly as §6.2.2 specifies · optional prompt-payment discount
terms. Never invoice ahead of the DEAF — §6.2.1 sequences acceptance first.

## 8. Cash-flow notes

- 10% of every development milestone is retained — model working capital assuming
  release no earlier than Final Acceptance (9.3).
- Weight D toward mid-project milestones (Tasks 3, 5, 6, 8) so revenue tracks
  effort; Tasks 1–2 amounts land early but are small by design.
- Recurring lines (10.1/11.1) begin at production deployment (8.6/8.7) — state
  the start trigger in the Assumptions column so there's no ambiguity about
  billing during UAT.
